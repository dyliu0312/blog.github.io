Fixing EasyConnect Startup Issues on Ubuntu

Fixing EasyConnect Startup Issues on Ubuntu

Problem: After installing EasyConnect on newer Ubuntu versions (20.04, 22.04, or later), the application often fails to launch.

1. Install EasyConnect

Download the .deb package:

Install with apt (which handles dependencies better):

At this point, you won’t able to run EasyConnet service succefully.

2. Identify the confliction

Acoording to the references, it seems to blame the old version of Electron framework, that’s nothing we can do. But we can downgrade the pango libraries to make it works.

Check which pango libs are linked:

Modern Ubuntu ships with newer libararies, you’ll likely see three libraries like libpangoft2-1.0.so.0, libpangocairo-1.0.so.0, and libpango-1.0.so.0.

Instead of modifying system-wide libraries (which could break other apps), we place the older pango versions directly inside EasyConnect’s folder. The application loads these local libraries first, bypassing the incompatible system versions.
